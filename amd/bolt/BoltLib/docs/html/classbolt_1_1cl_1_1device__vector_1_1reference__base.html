Detailed Description

template<typename T>
template<typename Container>
class bolt::cl::device_vector< T >::reference_base< Container >

A writeable element of the container The location of an element of the container may not actually reside in system memory, but rather in device memory, which may be in a partitioned memory space. Access to a reference of the container results in a mapping and unmapping operation of device memory.

Note:
The container element reference is implemented as a proxy object.
Warning:
Use of this class can be slow: each operation on it results in a map/unmap sequence.

inline

A get accessor function to return the encapsulated device buffer for const objects. This member function allows access to the Buffer object, which can be retrieved through a reference or an iterator. This is necessary to allow library functions to set the encapsulated buffer object as a kernel argument.

Note:
This get function could be implemented in the iterator, but the reference object is usually a temporary rvalue, so this location seems less intrusive to the design of the vector class.
